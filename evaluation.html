Critical Evaluation

Critical evaluation of the project
— User interface / user experience (if applicable)
Though we finished all the must/should have requirements, the user interface of the web application could be further improved.

— Functionality
— Stability
— Efficiency
— Compatibility
Our app is designed only for Android smartphones that supports Google ARcore. Whilst there are no plans to create an iOS version, this could be something to think about in the future, as many people use iPhones and iPads. Our client had specified Android compatibility hence we didn't explore the iOS option.

— Project management

Future Work

If we had more time, one of the main areas we would like to improve on is the variety of models that could be uploaded to the web app. At the moment, only certain .fbx models can be uploaded successfully but if we could extend this, the web app would become more versatile.
In addition, we would like to link the web app to the online asset store where over 3000 free assets could be downloaded to instead of having a model already downloaded. To enhance the customisability of the interactions with the models, we would also like to add more custom scripts. At the moment, we only have drag-and-drop and 360 video interactions with objects in the AR portal.
As well as this, we would like to have added in analytics, so users e.g. doctors could see which models/objects patients interacted with the most, to see what worries patients the most. This use could be then extended into training junior medics and other generic usage.
