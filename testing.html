Testing

Here we talk about the tests we carried out as well as user acceptance.

Test Cases

Our testing strategy was to do manual tests as we found automated tests aren't applicable to our project. Here, we have our test cases:
Test 1: Enter valid username and password, click submit.
Expected Result:
The user is taken to the user page
User ID cookie created
Session key cookie created
Session key in users.json is updated
Initial Date Passed: 8th Jan

Test 2: Enter invalid username and password
Expected Result:
The user remains on the same page
User ID cookie not created
Session key cookie not created
users.json unchanged
Initial Date Passed: 8th Jan

Test 3: Navigate to user.php while not logged in
Expected Result:
The user is transferred to the logout page, then to the login page
Initial Date Passed: 8th Jan -> retested when we moved to the 3D editor

Test 4: Navigate to editor.php while not logged in
Expected Result:
The user is transferred to the logout page, then to the login page
Initial Date Passed: 8th Jan

Test 5: Navigate to user.php with invalid cookies
Expected Result:
The user is transferred to the logout page, then to the login page
The cookies are cleared
Initial Date Passed: 8th Jan

Test 6: Navigate to editor.php with invalid cookies
Expected Results:
The user is transferred to the logout page, then to the login page
The cookies are cleared
Initial Date Passed: 8th Jan

Test 7: Navigate to user.php with valid cookies
Expected Results:
Cookies are refreshed
Session key cookie is changed
Session key in users.json is changed
Initial Date Passed: 23rd Mar -> had to be retested as we added passcodes after first prototype

Test 8: Navigate to editor.php with valid cookies
Expected Results:
Cookies are refreshed
Session key cookie is changed
Session key in users.json is changed
Initial Date Passed: 15th Jan

Test 9: Navigate to logout.php with valid cookies
Expected Results:
The user is transferred to the login page
The cookies are cleared
Initial Date Passed: 15th Jan

Test 10: Navigate to logout.php with invalid cookies
Expected Results:
The user is transferred to the login page
The cookies are cleared
Initial Date Passed: 15th Jan

Test 11: Navigate to logout.php with no cookies
Expected Results:
The user is transferred to the login page
Initial Date Passed: 15th Jan

Test 12: Create a new world on user.php
Expected Results:
New world appears on page
New world is added to users.json under the specified user ID
New world .json file created in worlds folder
New world folder created in Theater folder
Initial Date Passed: 20th Apr -> passed after editing user.php

Test 13: Edit metadata for a world on user.php
Expected Results:
The world .json file is updated, no unchanged information is lost
Initial Date Passed: 20th Apr -> passed after editing user.php

Test 14: Share world on user.php to an invalid email ID
Expected Results:
Nothing Happens (Check sendmail log)
Initial Date Passed: 20th Apr

Test 15: Share world to a valid email ID
Expected Results:
Email received at specified email ID
Initial Date Passed: 20th Apr

Test 16: Share world to more than 20 email IDs
Expected Results:
Email received at first 20
Initial Date Passed: 20th Apr

Test 17: Enter a mix of valid and invalid email IDs into the share world field, click share.
Expected Results:
Only the email addresses have emails sent to them. (check sendmail log)
Initial Date Passed: 20th Apr

Test 18: delete a world
Expected Results:
The world is removed from users.json
The world .json is removed from worlds folder
The world directory removed from the Theater folder
Initial Date Passed: 23rd Mar -> retested after moving to 3D editor

Test 19: select the edit world feature
Expected Results:
User is taken to editor.php
Specified world loads (place a specific uploaded model in a recognizable position beforehand to confirm)
Portal frame is present at the middle of the far left edge of the grid.
Initial Date Passed: 23rd Mar

Test 20: edit the world and exit without selecting save
Expected Results:
Nothing happens.
Initial Date Passed: 23rd Mar

Test 21: edit the world and save
Expected Results:
The world .json file should be updated.
Initial Date Passed: 23rd Mar

Test 22: upload a new .fbx file
Expected Results:
The file should be added to the folder in the Theater folder.
The Assetbundler program should have been run. (make sure to clear previous asset bundle to confirm, also, check if a .meta file has been created for the new .fbx)
Initial Date Passed: 18th Mar -> failed in first prototype hence we moved to 3D editor

Test 23: load a .fbx file from the default selection
Expected Results:
The model should be placed at the center of the grid, facing away from the portal.
Initial Date Passed: 18th Mar

Test 24: load a .fbx file from the uploaded selection
Expected Results:
The model should be placed at the center of the grid, facing away from the portal.
Initial Date Passed: 18th Mar

Test 25: move, rotate and scale an object in all three dimensions, then click save
Expected Results:
The world .json file should have new values in all the x,y,z fields.
Initial Date Passed: 20th Apr

Test 26: load the world in the mobile app
Expected Results:
The world should appear as designed on the web-application.
Initial Date Passed: 20th Apr

All tests passed as of 20th April 2019

User Acceptance

Feedback from our testers was generally positive, such as the improved user interface as well as the intuitive process of creating and customising rooms. We have yet to show the final product to our client but they were pleased with our prototype in March. We plan to showcase the final product soon.
